# 分批抓取详细规范

## 抓取流程

### 1. 访问 URL
- 使用 `coze-web-fetch` skill 获取页面内容
- 初始抓取参数：`--format markdown`（获取完整文本和链接信息）

### 2. 评估信息充足性

信息充足必须同时满足：
1. 页面包含至少 2 条与主题**直接相关**的**完整文章正文**（非标题或摘要）
2. 这些内容的总字符数 > 400 字


**信息不足的典型表现**：
- 仅包含导航菜单或分类列表
- 仅包含"更多信息请查看XX"的简短提示
- 标题列表但未展示具体内容
- 页面主要是图片、视频，文字内容少于 200 字

**如信息不足，执行子网页深入查询。**

### 3. 子网页深入查询

#### 3.1 识别相关子网页
- **优先级从高到低**：
  1. 文章详情页链接（如 `/article/123`、`/news/456`）
  2. 通知公告类页面（URL 包含 notice、announcement、news）
  3. 主题相关关键词匹配（链接文本包含 `{topic}` 相关词汇）
  4. 内容区域内的链接（排除导航菜单、页脚）
  5. 时间戳较新的链接（从标题或 URL 中提取日期）

- **每层选择数量**：
  - 第 1 层（主页）：最多选择 5 个最相关的链接
  - 第 2 层（子页）：每个子页最多选择 3 个链接
  - 第 3 层（子子页）：每个页面最多选择 2 个链接

#### 3.2 控制查询深度
- 最多查询 3 层子网页（初始页面 → 子页面1 → 子页面2 → 子页面3）
- 每层最多选择指定数量的链接（如上所述）

#### 3.3 抓取子页面

**文章详情页抓取流程**：
1. 使用 `--format markdown` 抓取完整文章内容
2. 提取关键信息：
   - 标题（从页面 `<title>` 或第一个 h1 提取）
   - 发布日期
   - 正文内容
   - 文中提到的相关链接（如有）

**文章列表页抓取流程**：
1. 使用 `--format markdown` 抓取
2. 识别文章标题和链接
3. 按时间顺序选择前 5 篇最新文章
4. 对每篇文章进行深度抓取（回到详情页抓取流程）

**详情页与目录页区分**：
- 详情页：URL 包含文章 ID（如 `/article/123`）、显示完整文章内容
- 目录页：列出多个文章标题和链接，需进一步跳转

#### 3.4 记录跳转路径
- 如信息来自子页面，需记录从主页到最终信息页面的跳转路径
- 格式：使用 "->" 连接各层页面标题
- 示例："主页 -> 学工通知 -> 软件工程学院关于开展2025学年校级研究生捐赠奖学金评选工作的通知"

#### 3.5 三层深度查询流程

**第 0 层：主页（初始页面）**
- 抓取方式：`--format markdown`
- 判断：是否包含足够的相关信息
- 如不足，进入第 1 层

**第 1 层：主栏目页面**
- 目标：导航菜单中的主要栏目（如"通知公告"、"学生工作"、"学术动态"）
- 抓取方式：`--format markdown`
- 识别：文章链接、子栏目链接
- 选择：最多 5 个最相关的链接进入第 2 层
- 优先级：通知公告类 > 学生工作类 > 学术动态类

**第 2 层：子栏目或文章列表页**
- 目标：文章列表页、分类页面
- 抓取方式：`--format markdown`
- 识别：具体文章链接
- 选择：最多 3 个最相关的链接进入第 3 层
- 或：如果是文章详情页，直接抓取并记录
- 判断标准：文章标题包含 `{topic}` 相关关键词

**第 3 层：文章详情页**
- 目标：具体的通知公告文章
- 抓取方式：`--format markdown`
- 提取：完整内容、标题、日期、相关链接
- 记录：到最终的 JSON 对象中
- 如仍信息不足，可提取文中提到的其他相关链接继续查询（但不超过 3 层）

### 4. 信息整理与 URL 标注

**必须包含的字段**：
```json
{
  "dep": "学院名称",
  "url": "通知公告的原始链接",
  "title": "通知标题（从<title>提取，完整准确）",
  "abstract": "通知内容概要，300-500字，包含关键信息，如时间节点、举办场所、对象、要求等",
  "publish_date": "发布日期（如能提取，格式：YYYY-MM-DD）"
}
```

**重要规则**：
- 必须为每条信息附上来源 URL ，该 URL 必须是信息详情页面的 URL
- 如信息来自子页面，需同时标注该子页面的 URL
- 避免重复信息，同一内容只保留最详细版本的来源
- 如遇到 PDF/DOCX 等文档链接，记录其 URL 并标注"文档链接"
- `publish_date` 字段用于记录信息的发布时间，便于后续排序和对比
- `details` 字段用于保存更详细的内容，当 `abstract` 无法充分表达时使用
- 抓取完成前必须自检：
  - 我是否访问了至少一个子页面？ 是/否
  - 我抓取到的信息是完整的通知正文，还是仅仅标题/摘要？

### 5. 保存结果

- 使用 `scripts/helper.py` 中的 `append_to_jsonl` 函数
- 将生成的 JSON 对象追加到 `.temp/fetch_{timestamp}.jsonl`
- 每个成功抓取的 URL 都要生成一个 JSON 对象

### 6. 错误处理

**失败情况**：
- 超时（Timeout）
- 403 Forbidden
- 404 Not Found
- 其他访问错误

**错误记录格式**：
```json
{
  "url": "失败的URL",
  "error": "失败原因"
}
```

**处理原则**：
- 记录错误后继续处理下一个 URL
- 不要因为个别 URL 失败而停止整个抓取流程
- 所有 URL 都要尝试抓取（成功或失败都要记录）
