# 汇总对比详细规范

## 汇总流程

### 1. 读取抓取结果
- 读取整个临时 JSONL 文件：`.temp/fetch_{timestamp}.jsonl`
- 过滤掉错误记录（包含 "error" 字段的记录）

### 2. 提取对比维度

**为每个非错误记录提取以下维度**：
- **学院**：从 title 或 URL 推断
- **与主题的相关性**：高/中/低
- **核心观点/事实**：一句话总结
- **独特信息**：其他页面没有提及的内容
- **数据/统计**：如有（数字、比例、金额等）

### 3. 交叉对比分析

#### 3.1 找出共同点
- 分析所有实体在"核心观点"上的共同主题
- 识别各学院在相关主题上的一致性

#### 3.2 列出明显差异
- 对比不同学院的侧重点（如某学院强调 AI，另一学院强调材料科学）
- 识别各学院的特色和优势领域

#### 3.3 识别矛盾信息
- 检查不同来源是否存在冲突信息
- 如发现矛盾，在总结中标注出来

### 4. 生成摘要对象

**必须包含的字段**：
```json
{
  "dep": "学院名称",
  "url": "信息的原始链接",
  "abstract": "核心摘要",
  "diff": "不同之处",
  "special": "独特信息"
}
```

**字段说明**：
- `dep`：学院名称（从 title 或 URL 推断）
- `url`：原始链接
- `abstract`：核心摘要（包含关键信息和主要特点）
- `diff`：该学院与其他学院的主要差异
- `special`：该学院的独特信息或亮点

**重要原则**：
- 每一项结果都要生成 JSON 对象
- 即使已存在相同的学院也要执行（确保每个 URL 都有对应记录）

### 5. 保存汇总结果

- 使用 `scripts/helper.py` 中的 `append_to_jsonl` 函数
- 将生成的 JSON 对象追加到 `.temp/summaries_{timestamp}.jsonl`
- 每个成功的记录都要生成一个摘要对象

## 对比维度说明

### 相关性判断标准
- **高**：内容与主题高度相关，包含详细信息、数据或政策文件
- **中**：内容与主题相关，但信息不够详细或全面
- **低**：内容与主题关联性较弱，或信息稀少

### 核心观点/事实
- 一句话概括该学院在该主题上的主要立场或特点
- 突出最关键的信息
- 避免细节，抓住本质

### 独特信息识别
- 与其他学院相比，该学院独有的信息
- 该学院特别强调或突出的特点
- 该学院在该主题上的特色优势

### 数据/统计
- 任何可量化的信息
- 数字、比例、金额、时间等
- 用于支撑核心观点的事实依据
